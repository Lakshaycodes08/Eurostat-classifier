Run AI-driven actions locally with deterministic behavior, policy control, and zero implicit side effects.

Install Docs GitLab Wiki

What problem we solve

AI tools can plan, but they shouldn’t execute freely. Swytchcode is the execution authority: no hidden side effects, no agent improvisation.

  • Deterministic execution – Same inputs → same outputs; no prompt-based execution.
  • Policy-driven control – tooling.json is the single source of truth for what is allowed.
  • Local-first – No runtime registry calls; exec uses only local config and integrations.
  • Tooling.json as contract – Human-readable, machine-enforced.

Who it’s for

  • Platform engineers and security-conscious teams
  • AI-assisted dev workflows (Cursor, Claude)
  • Teams that want explicit boundaries between planning and execution

What makes it different

  • No “agent autonomy” – agents call swytchcode exec; they don’t run arbitrary code.
  • No prompt-based execution – inputs and outputs are JSON; failures are explicit.
  • Explicit CLI boundary – swytch exec is the only integration entrypoint.
  • Human-readable + machine-enforced policies.
