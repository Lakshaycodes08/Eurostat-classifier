Security & Trust

Swytchcode treats AI as a planner, not an executor. Execution is explicit, local, and policy-bound.

Local execution

Tools run on your machine. No execution is delegated to remote services. Network calls happen only when a tool is explicitly defined to do so (e.g. an HTTP method).

No runtime registry

swytchcode exec uses only tooling.json and local integration bundles under .swytchcode/integrations/. There is no “phone home” during execution.

tooling.json is the single source of truth

Only tools listed in tooling.json can be executed. Human-readable, machine-enforced. No hidden allowlists; no policy interpretation by agents.

No hidden retries

Failures surface as exit codes and structured JSON. No silent retries. No agent-side interpretation of policy – if a tool isn’t in tooling.json or an integration isn’t found, exec fails explicitly.

Explicit failure

“Integration not found” and “tool not in tooling.json” are hard errors. Agents are expected to stop and report, not improvise.
