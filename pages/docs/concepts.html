Concepts

Deeper dive into how Swytchcode works. For a quick pass, see Overview.

Execution model

swytchcode exec is the only integration entrypoint. Editors and agents must not interpret policy or retry on their own; they call exec and respect its exit codes and JSON output.

tooling.json spec

Defines integrations (which bundles and versions) and tools (which canonical IDs are enabled). Schema and semantics are documented in the Wiki → tooling.json Spec.

Determinism guarantees

Same inputs to exec produce the same observable behavior. No prompt-based branching; no implicit retries. Output is JSON or raw as requested.

Error model

Errors are explicit: exit codes and structured error payloads. “Tool not found” and “integration not installed” are fatal; agents should not fall back to alternative execution paths.
